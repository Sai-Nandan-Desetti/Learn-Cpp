A copy constructor that copies the contents of other array into this array.

  • The array is default initialized if other.m_size is non-zero.
    • We do not need to zero-initialize the array because its contents are anyway going to be overwritten immediately.
  • And the this.m_size elements from other are copied to this.

IntArray & IntArray::operator= ( IntArray  other)

Overload the assignment operator to allow array assignments.

Parameters
otheran IntArray
Returns
IntArray& a reference to this IntArray

There are several things to note:

  • other is passed by value.
    • This ensures that a copy constructor is invoked by the compiler.
    • If, for some reason, the copy fails, control never enters the funtion body; therefore, this isn't modified.
  • We swap *this and other. This has the following benefits
    • We don't need to explicitly handle self-assignment.
    • swap is atomic.
    • Because a copy is already created, we do not need to re-copy the entire contents of the arrays.
    • We've elegantly avoided duplicating the code of the copy constructor.
  • After the swap, other points to the old content of *this. Once the control exits the function, the destructor of other is called. And it, thereby, deletes (i.e. invokes delete[] for) the array that was previously a member of *this.
    • Thus, we've avoided duplicating the code for the destructor as well!
  • Ref: [What is the copy-and-swap idiom?](https://stackoverflow.com/questions/3279543/what-is-the-copy-and-swap-idiom#3279550)
